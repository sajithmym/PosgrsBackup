# PostgreSQL Backup and Restore Desktop App

## Summary

This project is a PyQt5 desktop application for backing up and restoring PostgreSQL database tables. The app asks for editable connection values, database name, and then lets the user create a structured backup folder or restore from a previously generated backup folder.

## Architecture

- `main.py` is the root launcher for local use.
- `src/pg_backup_app/main_window.py` owns the PyQt5 user interface, validation prompts, threading, and status display.
- `src/pg_backup_app/db.py` owns PostgreSQL connection configuration and SQL identifier helpers.
- `src/pg_backup_app/backup_service.py` owns backup, restore, table discovery, DDL generation, CSV export, and SQL insert export.

The UI layer does not directly run SQL. It sends validated connection settings to `BackupService`, which keeps database and filesystem behavior in one service boundary.

## Setup

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
python main.py
```

## Default Connection Values

- Host: `localhost`
- Port: `5432`
- User: `postgres`
- Password: empty
- Database: user must enter this

The fields are auto-filled where defaults exist, and the user can change them before running backup or restore.

## UI Features

- System, Light, and Dark themes.
- Theme preference is saved with `QSettings` and restored when the app opens again.
- Non-secret workspace values are remembered: host, port, user, database, backup save folder, restore folder, and last completed backup folder.
- Passwords are not saved.
- `Test Connection` checks credentials before backup or restore.
- `Open Last Backup` opens the most recent successful backup folder.
- `Copy Status` copies the on-screen status history.
- The status panel is taller and uses a monospace font for easier reading.

## Backup Output

When the user chooses a save folder, the app creates a timestamped backup folder:

```text
<database>_backup_<timestamp>/
  csv/
    <schema>.<table>.csv
  sql/
    <schema>.<table>.sql
  table_creation_sql/
    create_tables.sql
  manifest.json
```

The CSV files are intended for inspection and restore. The SQL files contain table data as `INSERT` statements for external/manual usage. `create_tables.sql` creates schemas, tables, constraints, and non-constraint indexes. Generated columns are included in the table creation SQL but excluded from CSV/INSERT data so PostgreSQL can calculate them during restore.

## Restore Behavior

Restore asks for a backup folder generated by this app. It uses:

- `manifest.json` to map files back to schemas, tables, and columns.
- `table_creation_sql/create_tables.sql` to create schemas and tables.
- `csv/*.csv` to load table data back into the selected target database.

The restore runs in a database transaction. If an error occurs, the app rolls back the restore.

## Trade-offs and Risks

- Large databases can produce large SQL files and may take time to restore from CSV.
- Foreign-key cycles can still require manual handling if PostgreSQL rejects load order.
- The generated table SQL covers common tables, columns, defaults, identity/generated columns, constraints, and indexes, but it is not a full replacement for `pg_dump` for advanced objects such as triggers, views, functions, policies, extensions, publications, or grants.
- Passwords are never saved by the app.

## Manual Verification

1. Start the app with `python main.py`.
2. Enter valid PostgreSQL connection details and database name.
3. Optionally click `Test Connection`.
4. Choose a save folder, click `Backup Database`, and confirm the backup folder contains `csv`, `sql`, `table_creation_sql`, and `manifest.json`.
4. Create or choose an empty target database.
5. Enter the target database connection details.
6. Choose the generated backup folder, click `Restore Database`, and confirm tables/data are restored.
